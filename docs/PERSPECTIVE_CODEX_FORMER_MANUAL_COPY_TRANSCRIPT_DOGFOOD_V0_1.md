# Perspective Codex Former Manual Copy Transcript Dogfood v0.1

This document describes the pure local transcript dogfood harness that follows
PR #480, which added the Manual Codex Former Draft Copy Packet.

The intended dogfood path is:

1. Perspective Formation Input Bundle
2. CodexPerspectiveFormerInputPacket
3. CodexPerspectiveFormerDraftPromptContract
4. Manual Codex Former Draft Copy Packet
5. human-started Codex prompt
6. real Codex response transcript
7. extracted CodexPerspectiveCandidateDraft
8. prompt-contract fit evaluation
9. validateAndNormalizeCodexPerspectiveCandidateDraft
10. optional Worker-Facing Perspective Guidance

No real transcript was supplied for this PR. The harness therefore records the
main real-transcript scenario as BLOCKED. It does not fabricate a real Codex
response transcript, and synthetic fixtures are used only as negative/control
fixtures.

This harness is not Codex execution, not proof/evidence/readiness, not approval,
not merge authority, and not a Core decision.

## What This Adds

The transcript dogfood harness prepares the local evaluation path for a future
bounded, sanitized, human-supplied Codex response transcript. It builds the same
manual copy packet from PR #480-like bounded material, records transcript
provenance fields, attempts extraction, evaluates prompt-contract fit when a
draft exists, runs local validation, and reports whether downstream guidance can
be exercised.

The current report conclusion is BLOCKED because real transcript provenance is
missing.

## Real Transcript Requirements

A future real transcript fixture must be manually captured from a user-started
Codex session using the copyable prompt generated by the manual copy packet.
The fixture must be bounded, reviewable, sanitized, and labeled as a real manual
Codex former response transcript.

Required provenance should include:

- transcript kind and version;
- source manual copy packet id;
- bounded prompt hash or bounded prompt excerpt hash;
- captured by human manual action;
- captured time and Codex surface label when available;
- confirmation that the prompt came from the manual copy packet;
- bounded response text or extracted JSON text;
- extracted CodexPerspectiveCandidateDraft when available;
- redaction notes;
- false privacy and authority flags.

The fixture must not include hidden reasoning, private account data, browser
tokens or cookies, raw page dumps, raw PR diffs, raw review payloads, provider
logs, billing/token/OAuth payloads, API keys, secrets, or generated raw model
payload outside the bounded response needed for validation.

## Current Scenarios

`real_transcript_ready_or_needs_review` is blocked because no real
human-started Codex response transcript was supplied.

`transcript_extraction_failure_case` uses a synthetic negative control with no
parseable CodexPerspectiveCandidateDraft JSON. It must fail closed without
candidate-compatible review material.

`real_or_control_bad_response_regression` uses a clearly labeled synthetic bad
response control. It verifies that prompt-contract fit flags the response and
local validation blocks malformed or authority-claiming output.

`downstream_guidance_compatibility` is skipped with follow-up because the main
real transcript did not produce candidate-compatible review material.

## Browser/Computer-Use Validation
