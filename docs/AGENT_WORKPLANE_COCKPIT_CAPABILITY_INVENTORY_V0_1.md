# Agent Workplane Cockpit Capability Inventory v0.1

## Status And Scope

Status: Agent Workplane Cockpit Capability Inventory v0.1.

Scope: static inventory only. This document records useful legacy Cockpit
capabilities, why agents use them, whether Agent Workplane already has a native
replacement, and what had to remain accounted for until native replacement and
validation existed.

This PR adds no GuideBrief debug panel, no GuideBrief intent projection, no
Workplane node or panel contract, no Runner or DeltaBatch Workplane
integration, no runner behavior, no external actuation, no provider/OpenAI
call, no GitHub execution behavior, no Codex execution behavior, no durable
memory apply, and no Perspective apply.

Legacy Cockpit was not removed until native replacement and validation existed.
Cockpit Route Removal v0.1 later removed `/cockpit`,
`components/augnes-cockpit.tsx`, and the Workplane compatibility pointer after
`unique_useful_cockpit_capability_count: 0` was verified.

Runner / Workplane Metrics v0.1 is the metric-informed checkpoint before any
Legacy Cockpit Shrink Plan. Metrics are signals, not deletion authority:
`docs/AUGNES_WORKFLOW_METRICS_V0_1.md` measures native Workplane, GuideBrief,
runner DeltaBatch, stale/fallback, Cockpit absorption, and dogfood readiness
without deleting or shrinking legacy Cockpit functionality.

Legacy Cockpit Shrink v0.1 is documented in
`docs/AGENT_WORKPLANE_LEGACY_COCKPIT_SHRINK_V0_1.md`, with prior gate context
in `docs/AGENT_WORKPLANE_LEGACY_COCKPIT_SHRINK_PLAN_V0_1.md`. It uses this
inventory as the capability baseline, removes the full Cockpit mount from
`/workbench`, and originally preserved full compatibility at `/cockpit` without
deleting `components/augnes-cockpit.tsx`. Cockpit Route Removal v0.1 later
removed the route/component after readiness verification. Native absorption of
local-write/manual controls still requires a separate authority contract.

Legacy Cockpit Local UI Control Classification v0.1 is documented in
`docs/AGENT_WORKPLANE_LEGACY_COCKPIT_LOCAL_CONTROL_CLASSIFICATION_V0_1.md`.
It refines `legacy_local_ui_controls` into read-only, copy/export,
preview/local-draft, local-write, forbidden, compatibility-only, and
unknown/manual-review buckets. It is evidence/signaling only, not shrink
authority. Local-write controls remain blocked until a separate authority
contract exists; they are not kept alive by a retained Cockpit product route.

Allowed status values:

- `native_complete`
- `partially_native`
- `migrated_or_blocked_after_readiness`
- `obsolete`
- `remove_later`
- `needs_native_absorption`

## Post-Removal Coverage Rule

Any historically partial or pending capability must stay represented in Blank
State, Agent Workplane, Workplane State Proposal Review, Manual Controls
Migration rows, blocked-authority classifications, or obsolete/delete
classifications after Cockpit Route Removal v0.1. Native replacement must keep:

- source-backed fields or documented fixture fallback;
- visible authority and source/fallback copy;
- smoke coverage for reachability and forbidden authority;
- no hidden provider/OpenAI/GitHub/Codex execution;
- no runner behavior change unless the later scoped runner-read PR explicitly
  asks for it;
- no durable memory apply and no Perspective apply.

Legacy Cockpit was removed only after native replacement and validation verified
`unique_useful_cockpit_capability_count: 0`.
