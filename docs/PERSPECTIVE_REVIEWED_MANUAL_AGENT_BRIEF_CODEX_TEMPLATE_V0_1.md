# Perspective Reviewed Manual Agent Brief Codex Template v0.1

## Purpose and Scope

This PR adds a reviewed Codex prompt template builder around the refined Perspective Agent Brief `codex_handoff` packet.

It follows PR #456, which refined the Agent Brief handoff packet copy so the packet is context for a user-approved Codex PR workflow rather than execution authority by itself.

This is a local template, dogfood, and report slice. It is for human-reviewed copy/paste. It does not execute Codex, call GitHub, add routes, add UI, persist data, add DB or graph writes, call providers/models/APIs, implement OAuth, or add new source ingress.

## Core Model

The Agent Brief handoff packet is context.

The reviewed Codex prompt template is the human-reviewed instruction wrapper around that context.

Codex may code, test, and open a PR only when the surrounding user-approved prompt explicitly scopes that task. The template makes that workflow visible without granting merge, deploy, provider, persistence, Formation, or unscoped execution authority.

## PR-Centered Workflow

- Codex codes, tests, and opens a PR only when explicitly scoped by the user.
- ChatGPT reviews the PR.
- The user decides whether to merge.

Codex must not merge, deploy, publish, approve its own work, call external providers/models/APIs, call GitHub outside the scoped PR workflow, infer raw source content, request or expose secrets, persist source data, write DB/graph/proof/evidence/readiness state, treat the packet as Formation authority, or expand scope without user approval.

## Template Shape

`buildPerspectiveAgentBriefCodexPromptTemplate` accepts only a `perspective_agent_brief_handoff_packet.v0.1` packet whose audience is `codex_handoff`.

The returned template contains:

- `template_version: perspective_agent_brief_codex_prompt_template.v0.1`
- `intended_audience: codex`
- `workflow.codex_may`
- `workflow.codex_must_not`
- `workflow.review_chain`
- `source_packet`
- `prompt_text`
- `exclusions`

If a non-`codex_handoff` packet is passed to the builder, it throws locally. This keeps the prompt wrapper tied to the intended Codex PR workflow packet.

## Codex May

- Inspect the repository.
- Make scoped code, doc, or test changes only when the prompt explicitly asks for that task.
- Run relevant tests.
- Open a PR.
- Report changed files, tests, blockers, and risks.

## Codex Must Not

- Merge, deploy, publish, or approve its own work.
- Call external providers/models/APIs.
- Call GitHub outside the scoped PR workflow.
- Infer raw source content.
- Request or expose secrets.
- Persist source data.
- Write DB, graph, proof, evidence, or readiness state.
- Treat the packet as Formation authority.
- Expand scope without user approval.

## Source Packet Inclusion

The prompt includes the refined `codex_handoff` packet text generated by `buildPerspectiveAgentBriefHandoffPacket`.

The source packet is already compact and excludes raw manual input and raw ingress/admission values. The prompt template does not dump the Agent Brief object or the ingress admission object.

## Raw-Value Exclusions

Generated prompt artifacts intentionally exclude:

- raw pasted text
- raw `ingress_admission` JSON
- raw Agent Brief JSON
- candidate id values
- source ref values
- pointer ref values
- actor ref values
- consent ref values
- bounded summary values
- existing product packet bodies
- FormationReceipt body
- provider/model/API/GitHub/Codex/OAuth/token/billing/private/generated payloads

Manual ingress packets continue to render the selected material placeholder:

`Summary: omitted for manual ingress packet.`

Sample fixture packet behavior is unchanged.
