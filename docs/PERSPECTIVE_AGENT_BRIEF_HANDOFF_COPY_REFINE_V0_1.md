# Perspective Agent Brief Handoff Copy Refine v0.1

## Purpose and scope

This PR refines the copy generated by `buildPerspectiveAgentBriefHandoffPacket` based on the manual Codex review-loop dogfood findings. It follows PR #455.

The packet remains a local, pure, read-only consumption artifact. The packet shape, `perspective_agent_brief_handoff_packet.v0.1` version, section order, builder inputs, and builder outputs remain stable.

## Dogfood finding addressed

The dogfood packet passed the safety checks overall, but the `codex_handoff` copy said “Do not execute Codex.” That was safe but ambiguous because the packet is meant to be copied into a user-approved Codex PR workflow.

The reason why “Do not execute Codex” was replaced is that the old line blurred the difference between packet authority and a separately user-approved Codex task prompt.

This PR replaces that wording with clearer workflow semantics:

- The packet is context only.
- Codex may code, test, and open a PR only when the surrounding user prompt explicitly scopes that task.
- Codex must not merge, deploy, publish, or approve.
- ChatGPT reviews the PR.
- The user decides whether to merge.
- The packet does not grant execution, provider, persistence, GitHub mutation, or Formation authority by itself.

## Audience-aware copy

`codex_handoff` now describes a user-approved Codex PR workflow and states that coding/testing/opening a PR requires a surrounding scoped prompt.

`chatgpt_review` now describes ChatGPT and human review or planning context and explicitly avoids treating the packet as an implementation instruction by itself.

`agent_context` now describes read-only structured context for agent reasoning and states that it should not trigger side effects.

## Preserved exclusions

The packet still excludes raw pasted text, raw `ingress_admission` JSON, raw Agent Brief JSON, candidate/source/pointer/actor/consent values, bounded summary values, existing packet bodies, FormationReceipt body, external/private/provider/model/GitHub/Codex/OAuth/token/billing/prompt payloads, and raw source inference.

Manual ingress packets still render:

`Summary: omitted for manual ingress packet.`

Sample fixture packets without `ingress_context` may still include selected summaries.

## Out of scope

This PR does not add routes, modify `app/api`, add product UI, expose packets in product DOM, execute Codex, call GitHub, call providers/models/APIs, implement OAuth, implement ChatGPT Apps or Codex plugin integration, add DB schema or migrations, add persistence, add graph DB behavior, add proof/evidence/readiness writes, change graph topology, change node ids or types, change edge ids or types, change Event Rail structure, or change existing Perspective Handoff packet section order.

It preserves graph topology, node ids and types, edge ids and types, Event Rail structure, and existing Perspective Handoff packet section order.
